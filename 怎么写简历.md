# 专业技能

-  诚实：确保写上去的每一项，面试中都能回答上。
-  定制化：按照投递岗位的要求写，包含JD关键词 。
-  分类呈现：按类分层体现。例如编程语言、Android组件、UI、三方框架、网通通信、数据存储、并发异步、架构模式等
-  结果导向：将技能与“能力”或“效果”挂钩（熟悉内存优化 -> 掌握内存优化技巧，能使用 Profiler 分析并解决内存泄漏和 OOM 问题）
-  掌握程度：知道 → 了解；只是会用 → 熟悉；常用&理解原理 → 熟练使用；深入理解源码与思想 → 精通。



# 工作经历

-  意义：职责与贡献，突出你的角色和影响力。

-  倒序排列：最近的经历在最前面。

-  使用强力动词开头：

   -  管理类：指导、统领、管理、协调、主持、评估、整合

   -  成就类：实现、提升、降低、增加、节省、扩大、优化、加速、扭转、超额完成

   -  创造类：设计、开发、编写、构思、制作

   -  研究类：分析、评估、识别、诊断

   -  沟通类：协调、汇报、调解、促进

   -  执行类：执行、简化、安排、维护、落实



# 项目经验

-  意义：过程与成果，面试官看的是你解决问题的能力、产出、以及你在项目里承担了什么，而非一味看你名下有多少流行框架 。

-  定制化：按照投递岗位的要求写，包含JD关键词

-  写作格式：为了解决 [业务/技术问题] ，采用了[具体的技术/架构] ，通过 [具体的技术行动] ，实现了 [可量化的业务/技术成果] 

-  质与量互补：没什么亮点就多写经验，经验不够丰富就精写质量，给人一种有能力感。

-  使用强力动词开头：避免使用“负责、参与”这类无法说明你在项目中主次位置的动词，而要使用诸如“主导、独立完成”等词。
